# i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the frontend respect the user's language preference (pt-BR / en) using `next-intl` in non-routing mode.

**Architecture:** `next-intl` reads a `locale` cookie server-side, loads the matching JSON from `messages/`, and provides translations via `NextIntlClientProvider`. All `'use client'` pages/components use `useTranslations('Namespace')`. Language changes set the cookie and call `router.refresh()`.

**Tech Stack:** next-intl, Next.js 14 App Router, TypeScript, cookies

- [ ] **Step 2: Create i18n request config**

Create `frontend/src/i18n/request.ts`:

```ts
import { getRequestConfig } from 'next-intl/server';
import { cookies } from 'next/headers';

export default getRequestConfig(async () => {
  const store = await cookies();
  const locale = store.get('locale')?.value || 'en';
  const safeLocale = ['en', 'pt-BR'].includes(locale) ? locale : 'en';

  return {
    locale: safeLocale,
    messages: (await import(`../../messages/${safeLocale}.json`)).default,
  };
});
```
